## 1. 시작하기

### 1.1 시스템 요구사항

#### 최소 요구사항
- **운영체제**: Windows 10 이상 (64-bit)
- **Python**: Python 3.12 이상 (개발 환경의 경우)
- **메모리**: 4GB RAM 이상
- **디스크 공간**: 200MB 이상의 여유 공간
- **화면 해상도**: 1366x768 이상 권장

#### 권장 요구사항
- **운영체제**: Windows 11
- **메모리**: 8GB RAM 이상
- **화면 해상도**: 1920x1080 (Full HD)

### 1.2 설치 방법

#### 방법 1: 실행 파일 사용 (권장)

1. `auto_fm.exe` 파일을 다운로드합니다.
2. 원하는 폴더에 복사합니다.
3. `auto_fm.exe`를 더블클릭하여 실행합니다.

> **참고**: 처음 실행 시 Windows Defender 경고가 나타날 수 있습니다. "추가 정보" → "실행" 버튼을 클릭하면 실행됩니다.

![Windows Defender 경고](screenshots/01_windows_defender.png)

#### 방법 2: Python 소스 코드 실행

Python 개발 환경이 설치되어 있는 경우:

```bash
# 1. 프로젝트 폴더로 이동
cd C:\Users\[사용자명]\Documents\auto_fm\auto_fm

# 2. 의존성 패키지 설치 (최초 1회)
pip install -r requirements.txt

# 3. 프로그램 실행
python -m src.main
```

## 2. 기본 사용법

### 2.0 원가 계산의 이해

#### 2.0.1 원가 계산이란?

원가 계산은 특정 프로젝트나 서비스를 수행하는 데 필요한 총 비용을 체계적으로 계산하는 과정입니다.

**원가의 구성 요소**:

```
┌─────────────────────────────────────────┐
│           총 원가 (총액)                  │
├─────────────────────────────────────────┤
│                                         │
│  ① 노무비 (인건비)                       │
│     - 기본급                            │
│     - 제수당 (주휴, 연차)               │
│     - 퇴직금                            │
│     - 4대보험 (산재, 국민연금, 고용, 건강)│
│                                         │
│  ② 경비                                 │
│     - 운영비 (차량, 피복, 통신 등)      │
│     - 관리비 (교육, 복리후생 등)        │
│     - 시설비 (유지보수 등)              │
│     - 기타경비 (보험료 등)              │
│                                         │
│  ③ 일반관리비                           │
│     - 총 인건비의 일정 비율 (예: 5%)    │
│                                         │
│  ④ 이윤                                 │
│     - (인건비+경비+일반관리비)의        │
│       일정 비율 (예: 15%)                │
│                                         │
└─────────────────────────────────────────┘
```

**계산 공식**:
```
총액 = 노무비 + 경비 + 일반관리비 + 이윤

여기서,
일반관리비 = 노무비 × 일반관리비율
이윤 = (노무비 + 경비 + 일반관리비) × 이윤율
```

#### 2.0.2 시나리오의 개념

"시나리오"는 하나의 완전한 견적 또는 계산 케이스를 의미합니다.

**시나리오의 구성 요소**:
- 시나리오명 (식별자)
- 기준년도 (임금 기준)
- 직무별 인원 구성
- 경비 항목 및 금액
- 계산 결과

**시나리오 활용 예시**:

| 시나리오 | 기준년도 | 인원 | 월 총액 | 용도 |
|---------|---------|------|---------|------|
| A안_최소인원 | 2024 | 10명 | 30,000,000원 | 최소 인력 구성 |
| B안_표준인원 | 2024 | 15명 | 45,000,000원 | 표준 인력 구성 |
| C안_증원 | 2024 | 20명 | 60,000,000원 | 인력 증원 시 |
| 2025년_예상 | 2025 | 15명 | 48,000,000원 | 내년 예산 편성 |

**시나리오 관리 전략**:
1. **옵션별 관리**: 다양한 인력 구성 옵션을 시나리오로 저장
2. **연도별 관리**: 년도별 예산을 시나리오로 저장하여 비교
3. **버전별 관리**: 같은 프로젝트의 개정 이력을 버전으로 관리

### 2.2 기본 워크플로우

원가 산정 작업은 다음 5단계로 진행됩니다:

```
[1단계]           [2단계]           [3단계]           [4단계]           [5단계]
기준년도 선택  →  시나리오 생성  →  데이터 입력  →  집계 실행  →  결과 확인/저장
```

#### 1단계: 기준년도 선택
상단 도구 모음에서 임금 기준이 될 연도를 선택합니다.

**[스크린샷 4: 기준년도 선택 드롭다운]**
![기준년도 선택](screenshots/04_year_selection.png)

- **2023년**: 2023년 직무확정 기준 임금
- **2024년**: 2024년 직무확정 기준 임금
- **2025년**: 2025년 직무확정 기준 임금
- **2026년**: 2026년 직무확정 기준 임금

> **참고**: 기준년도를 변경하면 모든 직무의 일급 단가가 해당 연도 기준으로 자동 업데이트됩니다.

#### 2단계: 시나리오 생성
"시나리오명" 필드에 작업할 시나리오의 이름을 입력합니다.

```
예시:
- "A동 1차 시나리오"
- "B동 확정안"
- "옵션1_인원증원"
```

#### 3단계: 데이터 입력
- **직무별 인원**: 필요한 직무와 인원을 입력
- **경비**: 필요한 경비 항목과 금액을 입력

#### 4단계: 집계 실행
"집계 실행" 버튼을 클릭하여 원가를 계산합니다.

#### 5단계: 결과 확인/저장
- 우측 결과 영역에서 계산 결과 확인
- 필요 시 PDF/Excel로 내보내기
- "저장" 버튼으로 시나리오 저장

| 컬럼명 | 설명 | 입력 방식 | 예시 |

### 3.3 직무 추가 방법

#### 방법 1: 빈 행에서 직무 선택

1. 테이블의 빈 행을 클릭합니다.
2. "직무코드" 컬럼의 드롭다운을 클릭합니다.
3. 목록에서 원하는 직무를 선택합니다.

**[스크린샷 6: 직무코드 드롭다운 메뉴 펼쳐진 상태]**
![직무코드 드롭다운](screenshots/06_job_code_dropdown.png)

4. 직무를 선택하면 다음 정보가 자동으로 채워집니다:
   - **직무명**: 선택한 직무의 이름
   - **등급**: 직무 등급 (1급, 2급 등)
   - **일급(MD)**: 현재 기준년도의 임금 단가

#### 방법 2: Enter 키로 행 자동 추가

1. 마지막 행의 데이터 입력을 완료합니다.
2. **Enter** 키를 누릅니다.
3. 새로운 빈 행이 자동으로 추가됩니다.

### 3.4 인원 및 근무 조건 입력

직무를 선택한 후 다음 값들을 입력합니다:

#### ① 인원 (필수)
투입될 인원 수를 숫자로 입력합니다.

```
예시:
1      → 1명
2.5    → 2.5명 (시간제 근무 등)
10     → 10명
```

#### ② 월 근무일수 (필수)
월 평균 근무 일수를 입력합니다.

```
일반적인 값:
20.6   → 주 5일 근무 기준 (기본값)
26     → 주 6일 근무
30     → 월 근무
```

> **계산 방식**: 연간 근무일수 ÷ 12개월
> 예: 주 5일 근무 → (52주 × 5일) ÷ 12 = 21.67일 (실무에서는 20.6일 사용)

#### ③ 일 근무시간 (필수)
1일 근무 시간을 입력합니다.

```
일반적인 값:
8      → 전일제 근무 (기본값)
4      → 반일제 근무
6      → 6시간 근무
```

#### ④ 주휴일수 (선택)
월 평균 주휴 일수를 입력합니다.

```
계산 방식:
연간 주휴일수 ÷ 12개월 = 52주 ÷ 12 = 4.33일 (기본값)
```

#### ⑤ 연차일수 (선택)
월 평균 연차 일수를 입력합니다.

```
계산 방식:
연간 연차일수 ÷ 12개월 = 15일 ÷ 12 = 1.25일 (기본값)
```

### 3.5 입력 예시

#### 예시 1: 시설관리원 2명 투입

| 직무코드 | 직무명 | 등급 | 일급(MD) | 인원 | 월근무일 | 일근무시간 | 주휴일수 | 연차일수 | 월인건비 |
|---------|--------|------|----------|------|----------|-----------|---------|---------|----------|
| 10100 | 시설관리원 | 1급 | 110,000 | 2 | 20.6 | 8 | 4.33 | 1.25 | 5,500,000 |

**입력 순서**:
1. 직무코드 선택: "10100 - 시설관리원(1급)"
2. 인원 입력: 2
3. 나머지 값은 기본값 사용 (필요시 수정)
4. Enter 키로 확정

#### 예시 2: 미화원 3명, 시간제 근무

| 직무코드 | 직무명 | 등급 | 일급(MD) | 인원 | 월근무일 | 일근무시간 | 주휴일수 | 연차일수 | 월인건비 |
|---------|--------|------|----------|------|----------|-----------|---------|---------|----------|
| 10200 | 미화원 | 1급 | 95,000 | 3 | 20.6 | 4 | 4.33 | 1.25 | 2,850,000 |

**입력 순서**:
1. 직무코드 선택: "10200 - 미화원(1급)"
2. 인원 입력: 3
3. 일 근무시간 수정: 4 (반일제)
4. Enter 키로 확정

### 3.7 데이터 수정

이미 입력된 데이터를 수정하려면:

### 3.8 자동 계산

다음 값들은 입력한 데이터를 기반으로 자동 계산됩니다:

#### 월 인건비 계산 공식

```
월 인건비 = 일급(MD) × 인원 × 월 근무일수 × (일 근무시간 ÷ 8)
```

**예시**:
```
일급: 110,000원
인원: 2명
월 근무일수: 20.6일
일 근무시간: 8시간

월 인건비 = 110,000 × 2 × 20.6 × (8 ÷ 8)
         = 4,532,000원
```

시설 유지보수에 필요한 비용
- 시설유지비
- 장비유지비
- 안전관리비 등

#### 4. 기타경비
그 외 필요한 비용
- 보험료
- 제세공과금 등

### 5.4 계산 상세 공식

#### 인건비 계산 상세

각 직무의 인건비는 다음 항목으로 구성됩니다:

**1. 기본급**
```
기본급 = 일급 × 인원 × 월 근무일수 × (일 근무시간 ÷ 8)
```

**2. 주휴수당**
```
주휴수당 = 일급 × 인원 × 주휴일수 × (일 근무시간 ÷ 8)
```

**3. 연차수당**
```
연차수당 = 일급 × 인원 × 연차일수 × (일 근무시간 ÷ 8)
```

**4. 퇴직금 (기본급+제수당의 1/12)**
```
퇴직금 = (기본급 + 주휴수당 + 연차수당) ÷ 12
```

**5. 4대 보험**

산재보험 (사업주 부담):
```
산재보험 = (기본급 + 제수당) × 0.9%
```

국민연금 (사업주 부담):
```
국민연금 = (기본급 + 제수당) × 4.5%
```

고용보험 (사업주 부담):
```
고용보험 = (기본급 + 제수당) × 1.15%
```

건강보험 (사업주 부담):
```
건강보험 = (기본급 + 제수당) × 3.545%
```

노인장기요양보험 (건강보험의 12.81%):
```
장기요양보험 = 건강보험 × 12.81%
```

임금채권보장보험:
```
임금채권보장 = (기본급 + 제수당) × 0.06%
```

석면피해구제분담금:
```
석면피해구제 = (기본급 + 제수당) × 0.004%
```

**6. 월 인건비 합계**
```
월 인건비 = 기본급 + 주휴수당 + 연차수당 + 퇴직금 + 4대보험 합계
```

#### 계산 예시

**입력 조건**:
- 직무: 시설관리원 1급
- 일급: 110,000원
- 인원: 2명
- 월 근무일수: 20.6일
- 일 근무시간: 8시간
- 주휴일수: 4.33일
- 연차일수: 1.25일

**계산 과정**:

```
1. 기본급
   = 110,000 × 2 × 20.6 × (8 ÷ 8)
   = 4,532,000원

2. 주휴수당
   = 110,000 × 2 × 4.33 × (8 ÷ 8)
   = 952,600원

3. 연차수당
   = 110,000 × 2 × 1.25 × (8 ÷ 8)
   = 275,000원

4. 퇴직금
   = (4,532,000 + 952,600 + 275,000) ÷ 12
   = 479,966원

5. 4대보험
   산재: (4,532,000 + 952,600 + 275,000) × 0.009 = 51,837원
   국민연금: (4,532,000 + 952,600 + 275,000) × 0.045 = 259,182원
   고용: (4,532,000 + 952,600 + 275,000) × 0.0115 = 66,244원
   건강: (4,532,000 + 952,600 + 275,000) × 0.03545 = 204,183원
   장기요양: 204,183 × 0.1281 = 26,156원
   임금채권: (4,532,000 + 952,600 + 275,000) × 0.0006 = 3,455원
   석면피해: (4,532,000 + 952,600 + 275,000) × 0.00004 = 230원

   4대보험 합계 = 611,287원

6. 월 인건비 합계
   = 4,532,000 + 952,600 + 275,000 + 479,966 + 611,287
   = 6,850,853원
```

### 5.5 결과 해석

#### 정상적인 결과
집계가 정상적으로 완료되면:
- 요약 영역에 모든 금액이 표시됩니다.
- 상세 내역에 각 항목별 계산 결과가 나타납니다.
- 금액이 0원이 아니어야 합니다.

#### 비정상적인 결과
다음과 같은 경우 재확인이 필요합니다:

**경우 1: 총 인건비가 0원**
- 원인: 직무별 인원입력에 데이터가 없음
- 해결: 직무를 추가하고 인원을 입력하세요

**경우 2: 총 경비가 0원**
- 원인: 경비 입력에 데이터가 없음
- 해결: 필요한 경비 항목을 추가하세요 (경비가 없는 경우 정상)

**경우 3: 일부 직무의 인건비가 0원**
- 원인: 인원이 0이거나 입력되지 않음
- 해결: 해당 직무의 인원을 확인하세요

### 5.6 재집계

데이터를 수정한 후 다시 집계하려면:

1. 입력 데이터를 수정합니다.

3. 새로운 계산 결과가 표시됩니다.

## 6. 시나리오 관리

### 6.1 시나리오란?

시나리오는 하나의 완전한 원가 산정 작업 단위입니다. 각 시나리오에는 다음 정보가 포함됩니다:

- 시나리오명
- 기준년도
- 직무별 인원 데이터
- 경비 데이터
- 계산 결과

### 6.2 시나리오 저장

현재 작업 중인 시나리오를 저장하려면:

**방법 1: 저장 버튼 사용**

1. 상단 도구 모음에서 **"저장"** 버튼을 클릭합니다.

4. **"저장"** 버튼을 클릭합니다.

**파일 형식**: JSON (.json)

> **주의**: 불러오기를 실행하면 현재 작업 중인 데이터가 모두 교체됩니다. 필요한 경우 먼저 저장하세요.

## 9. 고급 기능

### 9.1 전년 대비 비교

전년도 데이터와 현재 계산 결과를 비교하여 비용 증감을 분석할 수 있습니다.

#### 9.1.1 전년 기준 데이터 저장

현재 계산 결과를 전년 기준으로 저장하려면:

1. 집계를 실행하여 결과를 확인합니다.
2. 상단 메뉴에서 **"도구"** → **"전년 기준으로 저장"**을 선택합니다.
3. 확인 대화상자에서 **"확인"**을 클릭합니다.

> **참고**: 전년 기준 데이터는 시나리오별로 저장되며, 동일한 시나리오명으로 비교할 때만 적용됩니다.

#### 9.1.2 전년 대비 증감 확인

전년 기준 데이터가 저장된 상태에서 새로운 계산을 수행하면:

**요약 영역 변화**:

| 항목 | 올해 | 전년 | 증감액 | 증감률 |
|------|------|------|--------|--------|
| 총 인건비 | 10,000,000 | 9,500,000 | +500,000 | +5.3% |
| 총 경비 | 500,000 | 450,000 | +50,000 | +11.1% |
| 일반관리비 | 500,000 | 475,000 | +25,000 | +5.3% |
| 이윤 | 1,650,000 | 1,563,750 | +86,250 | +5.5% |
| **총액** | **12,650,000** | **11,988,750** | **+661,250** | **+5.5%** |

#### 9.1.3 증감 원인 분석

증감의 주요 원인을 파악하려면:

**인건비 증감 요인**:
1. **임금 단가 변동**: 기준년도 변경으로 인한 일급 인상
2. **인원 변동**: 투입 인원 수 증감
3. **근무 조건 변경**: 월 근무일수, 일 근무시간 변경
4. **보험료율 변경**: 4대보험료율 변동

**경비 증감 요인**:
1. **단가 변동**: 경비 항목별 단가 인상/인하
2. **수량 변동**: 필요 수량 증감
3. **항목 추가/삭제**: 새로운 경비 항목 추가 또는 삭제

#### 9.1.4 전년 대비 비교 활용 예시

**예시 1: 임금 인상률 반영**

전년: 2024년 기준, 시설관리원 1급 일급 110,000원
올해: 2025년 기준, 시설관리원 1급 일급 115,000원

```
인원: 2명
월 근무일수: 20.6일

전년 월 인건비: 110,000 × 2 × 20.6 = 4,532,000원
올해 월 인건비: 115,000 × 2 × 20.6 = 4,738,000원
증감액: +206,000원 (+4.5%)
```

**예시 2: 인원 증원 효과 분석**

전년: 미화원 2명
올해: 미화원 3명 (1명 증원)

```
일급: 95,000원
월 근무일수: 20.6일

전년 월 인건비: 95,000 × 2 × 20.6 = 3,914,000원
올해 월 인건비: 95,000 × 3 × 20.6 = 5,871,000원
증감액: +1,957,000원 (+50%)
```

#### 9.1.5 전년 기준 데이터 초기화

전년 기준 데이터를 삭제하려면:

1. **"도구"** → **"전년 기준 초기화"** 선택
2. 확인 대화상자에서 **"확인"** 클릭
3. 전년 대비 표시가 사라집니다

### 9.2 일반관리비/이윤 조정

기본 설정된 일반관리비(5%)와 이윤(15%) 비율을 프로젝트 특성에 맞게 조정할 수 있습니다.

#### 9.2.1 현재 비율 확인

현재 설정된 비율을 확인하려면:

1. 상단 메뉴에서 **"설정"** → **"일반관리비/이윤 설정"**을 클릭합니다.
2. 현재 비율이 표시됩니다.

**기본값**:
- 일반관리비: 총 인건비의 5%
- 이윤: (총 인건비 + 총 경비 + 일반관리비)의 15%

#### 9.2.2 비율 변경 방법

비율을 변경하려면:

**Step 1: 설정 대화상자 열기**

**Step 2: 비율 입력**

1. **일반관리비율** 필드에 원하는 비율 입력 (%)
   - 예: 5 (5%), 7 (7%), 10 (10%)
2. **이윤율** 필드에 원하는 비율 입력 (%)
   - 예: 15 (15%), 10 (10%), 20 (20%)

**Step 3: 저장 및 재집계**

1. **"저장"** 버튼 클릭
2. 메인 화면으로 돌아와 **"집계 실행"** 버튼 클릭
3. 새로운 비율이 적용된 결과가 표시됩니다

#### 9.2.3 비율 변경 효과 예시

**예시: 이윤율 15% → 10% 변경**

```
총 인건비: 10,000,000원
총 경비: 500,000원
일반관리비: 500,000원 (5%)

기존 이윤 (15%):
= (10,000,000 + 500,000 + 500,000) × 0.15
= 1,650,000원

변경 후 이윤 (10%):
= (10,000,000 + 500,000 + 500,000) × 0.10
= 1,100,000원

차이: -550,000원 (-33.3%)
총액 차이: -550,000원
```

#### 9.4.1 월 근무일수 설정

월 평균 근무일수의 기본값을 설정합니다.

**Step 1: 설정 메뉴 열기**

1. **"설정"** → **"기준 설정"** 메뉴 클릭
2. "기준 설정" 대화상자가 나타납니다

**Step 2: 월 근무일수 입력**

1. "월 근무일수" 필드에 원하는 값 입력
2. 일반적인 값:
   - **20.6일**: 주 5일 근무 (월~금)
   - **26일**: 주 6일 근무 (월~토)
   - **30일**: 월 근무 (휴일 없음)

**계산 방법**:
```
주 5일 근무: (52주 × 5일) ÷ 12개월 = 21.67일 → 실무: 20.6일
주 6일 근무: (52주 × 6일) ÷ 12개월 = 26일
```

**Step 3: 저장**

1. **"저장"** 버튼 클릭
2. 이후 새로 추가되는 직무 행에 자동으로 적용됩니다

#### 9.4.2 일 근무시간 설정

1일 근무시간의 기본값을 설정합니다.

**설정 방법**:

1. **"설정"** → **"기준 설정"** 메뉴
2. "일 근무시간" 필드에 값 입력
3. 일반적인 값:
   - **8시간**: 전일제 근무 (기본값)
   - **4시간**: 반일제 근무
   - **6시간**: 단시간 근무

**계산에 미치는 영향**:

일 근무시간은 월 인건비 계산에 직접 반영됩니다.

```
월 인건비 = 일급 × 인원 × 월근무일수 × (일근무시간 ÷ 8)

예시 (4시간 근무):
= 110,000 × 2 × 20.6 × (4 ÷ 8)
= 110,000 × 2 × 20.6 × 0.5
= 2,266,000원 (전일제의 50%)
```

**설정 방법**:
1. **"설정"** → **"기준 설정"** 메뉴
2. 각 필드에 원하는 값 입력
3. **"저장"** 클릭

모든 기본 설정을 초기값으로 되돌리려면:

##### Q11. 퇴직금 계산이 맞나요?

**A11. 퇴직금 계산 공식**
```
퇴직금 = (기본급 + 제수당) ÷ 12

예시:
기본급: 4,532,000원
제수당: 1,227,600원 (주휴+연차)
퇴직금 = (4,532,000 + 1,227,600) ÷ 12
      = 5,759,600 ÷ 12
      = 479,966원
```

이는 근로기준법에 따른 퇴직금의 월 적립분입니다.

### 11.3 계산 공식 요약

#### 인건비 계산

```
기본급 = 일급 × 인원 × 월근무일수 × (일근무시간 ÷ 8)

주휴수당 = 일급 × 인원 × 주휴일수 × (일근무시간 ÷ 8)

연차수당 = 일급 × 인원 × 연차일수 × (일근무시간 ÷ 8)

퇴직금 = (기본급 + 주휴수당 + 연차수당) ÷ 12

산재보험 = (기본급 + 주휴수당 + 연차수당) × 0.9%
국민연금 = (기본급 + 주휴수당 + 연차수당) × 4.5%
고용보험 = (기본급 + 주휴수당 + 연차수당) × 1.15%
건강보험 = (기본급 + 주휴수당 + 연차수당) × 3.545%
장기요양보험 = 건강보험 × 12.81%
임금채권보장 = (기본급 + 주휴수당 + 연차수당) × 0.06%
석면피해구제 = (기본급 + 주휴수당 + 연차수당) × 0.004%

월 인건비 = 기본급 + 주휴수당 + 연차수당 + 퇴직금 +
            산재 + 국민연금 + 고용 + 건강 + 장기요양 + 임금채권 + 석면피해
```

#### 총액 계산

```
총 인건비 = Σ(각 직무의 월 인건비)

총 경비 = Σ(각 경비 항목의 합계)

일반관리비 = 총 인건비 × 5%

이윤 = (총 인건비 + 총 경비 + 일반관리비) × 15%

총액 = 총 인건비 + 총 경비 + 일반관리비 + 이윤
```

#### 11.5.3 워크플로우 3: 옵션별 견적 비교

**상황**: 클라이언트가 3가지 인력 구성 옵션에 대한 견적을 요청했습니다.

**옵션**:
- A안: 최소 인력 (비용 최소화)
- B안: 표준 인력 (균형)
- C안: 충원 인력 (서비스 강화)

**Step 1: A안 (최소 인력) 작성**

인력 구성:
- 시설관리원 1급: 1명
- 미화원 1급: 1명
- 경비원 2급: 2명 (등급 낮춤)

1. 시나리오명: "A안_최소인력"
2. 위 인력 구성 입력
3. 집계 실행
4. 저장: `A안_최소인력.json`

**Step 2: B안 (표준 인력) 작성**

인력 구성:
- 시설관리원 1급: 1명
- 미화원 1급: 2명
- 경비원 1급: 3명

1. "신규" 클릭하여 화면 초기화
2. 시나리오명: "B안_표준인력"
3. 위 인력 구성 입력
4. 집계 실행
5. 저장: `B안_표준인력.json`

**Step 3: C안 (충원 인력) 작성**

인력 구성:
- 시설관리원 1급: 2명 (1명 증원)
- 미화원 1급: 3명 (1명 증원)
- 경비원 1급: 4명 (1명 증원)

1. "신규" 클릭
2. 시나리오명: "C안_충원인력"
3. 위 인력 구성 입력
4. 집계 실행
5. 저장: `C안_충원인력.json`

**Step 4: 3가지 옵션 비교**

1. "비교" 버튼 클릭
2. A안, B안, C안 모두 선택
3. "비교 실행" 클릭
4. 비교 결과 확인:

| 항목 | A안 | B안 | C안 |
|------|-----|-----|-----|
| 총 인건비 | 15M | 19M | 26M |
| 총액 | 18M | 24M | 32M |
| 증감 | 기준 | +6M | +14M |

5. "Excel 내보내기"로 상세 비교표 생성
6. 클라이언트에게 3가지 옵션 제시

#### 11.5.4 워크플로우 4: 일반관리비/이윤율 조정

**상황**: 클라이언트가 이윤율을 15%에서 10%로 낮춰달라고 요청했습니다.

**Step 1: 기존 시나리오 불러오기**

1. "불러오기" 버튼 클릭
2. 기존 시나리오 선택
3. 현재 결과 확인 (이윤 15%)

**Step 2: 이윤율 변경**

1. "설정" → "일반관리비/이윤 설정" 메뉴
2. 이윤율을 "15"에서 "10"으로 변경
3. "저장" 클릭

**Step 3: 재집계 및 비교**

1. 메인 화면으로 돌아와 "집계 실행" 버튼 클릭
2. 새로운 결과 확인:
   ```
   이전 (15%): 총액 23,578,503원
   변경 (10%): 총액 22,478,503원

   ```

**Step 4: 새로운 시나리오로 저장**

1. 시나리오명 변경: 기존명 + "_이윤10%"
2. "저장" 버튼으로 새 파일 저장
3. 클라이언트에게 수정 견적 제출

#### 11.5.5 워크플로우 5: 복잡한 경비 항목 관리

**상황**: 대형 빌딩 관리로 다양한 경비 항목을 관리해야 합니다.

**경비 항목 (15개 이상)**:
- 차량 관련: 휘발유, 정비비, 보험료
- 피복 관련: 하복, 동복, 안전화
- 소모품: 청소도구, 사무용품, 위생용품
- 교육: 직무교육, 안전교육
- 복리후생: 건강검진, 회식비
- 기타: 시설유지비, 안전관리비

**Step 1: 경비 입력 전략 수립**

**전략 A: 상세 입력**
- 모든 항목을 세부적으로 입력
- 장점: 정확한 비용 파악
- 단점: 입력 시간 증가

**전략 B: 그룹별 통합**
- 유사 항목을 묶어서 입력
- 장점: 빠른 입력
- 단점: 상세도 낮음

**권장: 전략 A (상세 입력)**

**Step 2: 경비 항목 그룹별 입력**

**그룹 1: 운영비 (차량, 피복, 통신 등)**
```
차량유지비 → 휘발유 (1,800원 × 300L)
차량유지비 → 정비비 (500,000원 × 2회)
피복비 → 하복 (50,000원 × 18벌)
피복비 → 동복 (70,000원 × 18벌)
```

**그룹 2: 관리비 (교육, 복리후생 등)**
```
교육훈련비 → 직무교육 (100,000원 × 5회)
복리후생비 → 건강검진 (80,000원 × 18명)
```

**그룹 3: 시설비**
```
시설유지비 → 소규모공사 (2,000,000원 × 1식)
안전관리비 → 안전점검 (300,000원 × 12회)
```

**Step 3: 경비 합계 확인**

1. 모든 경비 입력 완료 후 "집계 실행"
2. 경비 상세 탭에서 그룹별 소계 확인
3. 총 경비 확인

**Step 4: 경비 최적화**

비용이 예산을 초과하는 경우:
1. 각 항목의 필요성 재검토
2. 수량 또는 단가 조정
3. 우선순위 낮은 항목 삭제
4. 재집계 후 확인

```
1. 비교 결과를 Excel로 내보내기
2. 옵션별 장단점 분석 문서 작성
3. 관리자 승인
4. 확정 시나리오 저장 및 보관
```

**예산 관리 팁**:
- **5% 예비비 확보**: 예상치 못한 비용 대비
- **분기별 검토**: 3개월마다 실적과 비교
- **시나리오 버전 관리**: 수정 이력 추적 가능

#### 11.7.3 계약 담당자를 위한 가이드

**업무 시나리오**: 용역 계약 체결 및 갱신

**계약 준비 프로세스**:

**1단계: 계약 조건 확인**
```
확인 사항:
- 계약 기간 (1년, 2년 등)
- 임금 기준년도
- 일반관리비율 (계약서 명시)
- 이윤율 (계약서 명시)
- 월 근무일수 (주 5일/6일)
- 일 근무시간 (8시간/기타)
```

**2단계: 원가 계산**
```
작업:
1. 계약 조건에 맞게 프로그램 설정
2. 필요 인력 구성 입력
3. 예상 경비 입력
4. 집계 실행
```

**3단계: 계약 금액 산정**
```
고려 사항:
- 최저가 입찰 vs 적격 심사
- 경쟁사 견적 범위
- 수익성 (이윤율 10% 이상 확보)
- 리스크 요인 (인력 수급, 물가 변동 등)

조정:
1. 이윤율 조정으로 최종 금액 맞추기
2. 또는 인력 구성 변경 (등급 조정)
3. 경비 항목 최적화
```

**4단계: 계약서 작성 및 체결**
```
첨부 자료:
1. 원가 계산서 (PDF)
2. 인력 구성표 (Excel)
3. 경비 명세서 (Excel)
4. 시나리오 파일 (JSON) - 내부 보관용
```

**계약 갱신 시**:
```
절차:
1. 기존 계약 시나리오 불러오기
2. 기준년도만 변경 (임금 자동 인상)
3. 집계 후 증가액 확인
4. 클라이언트와 협의
5. 확정 후 새 시나리오로 저장
```

#### 11.7.4 회계 담당자를 위한 가이드

**업무 시나리오**: 원가 분석 및 손익 관리

**원가 분석 프로세스**:

**1단계: 계약 원가 vs 실제 원가 비교**
```
목적: 계약 시 산정한 원가와 실제 집행액 비교

작업:
1. 계약 시 시나리오 불러오기
2. 실제 지출 데이터를 별도 시나리오로 작성
3. 두 시나리오를 비교 화면에서 분석
4. 차이 발생 원인 파악
```

**차이 발생 원인 예시**:
- 추가 인력 투입 (교체, 증원)
- 경비 초과 (유류비 상승, 긴급 수리 등)
- 보험료율 변경
- 임금 소급 적용

**2단계: 손익 분석**
```
계산:
1. 계약 금액 - 실제 원가 = 실제 이윤
2. 실제 이윤율 = 실제 이윤 ÷ 계약 금액 × 100

예시:
계약 금액: 24,000,000원
실제 원가: 22,000,000원
실제 이윤: 2,000,000원
실제 이윤율: 8.3%

판단: 계획 이윤율(10%) 대비 1.7% 감소
```

**3단계: 개선 방안 수립**
```
이윤율 개선 방법:
1. 경비 절감 (불필요 항목 제거)
2. 인력 운영 효율화 (교대 근무, 겸직 등)
3. 차기 계약 시 이윤율 상향 조정
4. 프로세스 개선을 통한 관리비 절감
```

**4단계: 보고서 작성**
```
월간/분기별 보고서 항목:
1. 계획 대비 실적 비교표
2. 주요 증감 원인 분석
3. 손익 추이 그래프
4. 다음 기간 예측 및 개선 계획
```

**Excel 활용 팁**:
```
1. 프로그램에서 Excel로 내보내기
2. 피벗 테이블로 항목별 집계
3. 차트로 시각화
4. 월별 데이터 추가하여 트렌드 분석
```

### 11.9 고급 팁 모음

#### 11.9.1 계산 정확도 향상

**소수점 처리 팁**
```
인원 입력 시:
- 2.0명: 2명 전일제
- 2.5명: 2명 전일제 + 1명 반일제 (4시간)
- 2.75명: 2명 전일제 + 1명 6시간 근무 (6/8 = 0.75)
- 2.33명: 2명 전일제 + 1명 주 2일 근무 (2/6 = 0.33)

계산 방법:
시간제 근무 비율 = 실제 근무시간 ÷ 8시간
또는
시간제 근무 비율 = 실제 근무일수 ÷ 표준 근무일수
```

**반올림 규칙**
```
프로그램 내부 계산:
- 중간 계산: 소수점 이하 전체 유지
- 최종 표시: 원 단위에서 반올림

수동 검증 시:
1. 각 단계를 소수점까지 계산
2. 최종 단계에서만 반올림
3. 프로그램 결과와 비교
```

**정확도 검증 방법**
```
Step 1: 샘플 데이터로 수동 계산
- 간단한 케이스 선택 (인원 1명)
- 계산기로 단계별 계산
- 프로그램 결과와 비교

Step 2: 극단적 케이스 테스트
- 인원 0.1명 (시간제)
- 월 근무일수 30일 (최대)
- 일 근무시간 12시간
- 결과의 합리성 확인

Step 3: 총액 역산
- 총액에서 이윤 제거
- 일반관리비 제거
- 노무비 + 경비와 일치 확인
```

#### 11.9.2 효율적인 데이터 관리

**시나리오 버전 관리**
```
명명 규칙:
- v1, v2, v3... 버전 번호 사용
- 날짜 포함 (YYYYMMDD)
- 상태 표시 (_초안, _검토중, _확정)

예시:
A아파트_v1_20240201_초안.json
A아파트_v2_20240205_검토중.json
A아파트_v3_20240210_확정.json
```

**변경 이력 추적**
```
별도 문서 작성:
파일명: A아파트_변경이력.txt

내용 예시:
─────────────────────────────
v1 (2024-02-01):
- 초안 작성
- 인원: 시설관리원 1, 미화원 2, 경비원 3
- 총액: 23,500,000원

v2 (2024-02-05):
- 경비원 1명 감원 (3→2)
- 피복비 단가 조정 (50,000→45,000)
- 총액: 20,300,000원

v3 (2024-02-10):
- 이윤율 변경 (15%→10%)
- 총액: 19,100,000원 (확정)
─────────────────────────────
```

**Excel로 시나리오 이력 관리**
```
시트 구성:
Sheet1: 시나리오 목록
| 파일명 | 작성일 | 작성자 | 총액 | 상태 |
|--------|--------|--------|------|------|
| A아파트_v1 | 2024-02-01 | 홍길동 | 23,500,000 | 초안 |
| A아파트_v2 | 2024-02-05 | 홍길동 | 20,300,000 | 검토중 |

Sheet2: 비교표
| 항목 | v1 | v2 | v3 | 증감 |
|------|----|----|----| ---- |
```

#### 11.9.3 보고서 작성 팁

```
   - 파일명: [프로젝트]_비교_[날짜].pdf
```

```
3. 발표 자료 보완
   - 주요 숫자를 큰 폰트로 강조
   - 증감액을 화살표(↑↓)로 표시
   - 핵심 포인트를 색상으로 하이라이트
```

#### 11.9.4 트러블슈팅 체크리스트

**문제 발생 시 체크리스트**
```
□ Step 1: 프로그램 재시작
  - 종료 후 재실행
  - Windows 작업 관리자에서 프로세스 확인

□ Step 2: 파일 확인
  - JSON 파일 형식 오류 확인
  - 파일 인코딩 확인 (UTF-8)
  - 파일 권한 확인

□ Step 3: 데이터베이스 확인
  - cost_calc.db 파일 존재 확인
  - 파일 크기 확인 (0바이트 아닌지)
  - 필요 시 데이터베이스 재생성

□ Step 4: 로그 확인
  - logs/app.log 파일 열기
  - 최신 오류 메시지 확인
  - 구글 검색 또는 지원팀 문의

□ Step 5: 백업에서 복구
  - 최근 백업 파일 찾기
  - 백업 파일 복원
  - 작업 재시작
```

**자주 발생하는 문제별 해결법**
```
문제 1: "집계 실행 후 결과가 0원"
해결:
→ 직무코드 선택 확인
→ 인원이 0이 아닌지 확인
→ 기준년도 선택 확인

문제 2: "드롭다운이 비어있음"
해결:
→ 프로그램 재시작
→ cost_calc.db 파일 확인
→ 데이터베이스 재생성 고려

문제 3: "파일 저장 실패"
해결:
→ 저장 폴더 쓰기 권한 확인
→ 디스크 공간 확인
→ 파일명에 특수문자 제거

문제 4: "Excel 내보내기 실패"
해결:
→ Excel 프로그램 닫기
→ 동일 파일명 확인 후 삭제
→ 다른 폴더에 저장 시도
```
